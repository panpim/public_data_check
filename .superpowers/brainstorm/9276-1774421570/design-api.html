Design — Section 3: API Route & Services

POST /api/checks/run orchestrates the full pipeline in one request

POST /api/checks/run — step by step

1
Auth check — reject if no session (401)
2
Validate input — borrowerName required, driveFolderUrl must be a valid Drive URL
3
Look up provider — resolve providerKey → PublicCheckProvider from registry (throws 400 if unknown)
4
Run search — provider.runSearch(input) → NormalizedCheckResult (Playwright, ~10–25s)
5
Generate PDF — generateEvidencePdf(input, result, filename) → Buffer
6
Upload to Drive — extract folder ID from URL, upload PDF with user's OAuth token → get file ID + URL
7
Write audit row — prisma.searchRun.create(…) — always written even on partial failure
8
Return JSON — { status, resultsCount, summaryText, driveUrl, runId }

Drive service — drive.ts

// Uses the user's Google OAuth access token from NextAuth session
// scope: drive.file — can only touch files created by this app

uploadFileToDrive(
  accessToken: string,
  folderId: string,    // extracted from Drive URL
  filename: string,
  pdfBuffer: Buffer
) → { fileId, webViewLink }

Error handling

Failure point Behaviour
Playwright search fails Result status = error, audit row written, PDF still generated with error page
Drive upload fails Audit row written with no fileId, result still returned to UI with a warning
Drive token expired NextAuth refreshes automatically via account.refresh_token — 401 returned if refresh fails
