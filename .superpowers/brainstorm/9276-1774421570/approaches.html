Architecture Approach

Three ways to wire the Next.js app together

A ★

App Router + API Route

Next.js App Router (server components where possible), one dedicated POST /api/checks/run route that runs Playwright + pdf-lib + Drive upload as a single async job. Auth via NextAuth route handler. Recommended — matches the architecture doc exactly, modern, clean separation.

Pros

  • Matches spec exactly
  • Easy to add more providers later
  • Server components keep secrets server-side

Cons

  • Long-running route (~30s) — needs a non-serverless host
B

App Router + Server Actions

Replace /api/checks/run with a Next.js Server Action called directly from the form. No explicit API route. Slightly simpler form wiring.

Pros

  • Less boilerplate
  • Direct form → server function

Cons

  • Harder to call from external scripts
  • Less standard for long-running tasks
  • Deviates from the spec
C

Pages Router + API Route

Use the older Next.js Pages Router (pages/ directory). Everything works the same under the hood.

Pros

  • More familiar if team knows older Next.js

Cons

  • No server components
  • Less modern pattern
  • No benefit over A
