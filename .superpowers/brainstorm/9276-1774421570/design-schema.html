Design — Section 2: Database Schema

Prisma model for SQLite. One table — SearchRun — is the authoritative audit log.

// prisma/schema.prisma

datasource db {
  provider = "sqlite"
  url      = env("DATABASE_URL")
}

generator client {
  provider = "prisma-client-js"
}

model SearchRun {
  id                  String  @id @default(cuid())
  createdAt           DateTime @default(now())
  createdByEmail      String  // Google account email

  // — Search input —
  borrowerName        String
  borrowerIdCode      String?
  loanReference       String?
  providerKey         String  // "avnt_insolvency"
  driveFolderUrl      String

  // — Normalised result —
  resultStatus        String  // no_match | match_found | ambiguous | error
  resultsCount        Int
  matchedSummary      String?

  // — Drive evidence —
  uploadedFileId      String?
  uploadedFileUrl     String?

  // — Raw payloads for reprocessing —
  requestPayloadJson   String?
  normalizedResultJson String?
}

Key decisions

Decision Rationale
@id @default(cuid()) Collision-safe, URL-safe string IDs — no integer sequences to expose
resultStatus String SQLite has no enum type — stored as string, validated in app layer
requestPayloadJson Full snapshot enables re-running or re-generating the PDF without re-searching
No Account/Session tables NextAuth is configured for JWT sessions — no DB adapter needed, keeping the schema minimal
